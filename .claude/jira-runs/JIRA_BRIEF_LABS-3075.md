## Description

Summary
Self-hosted OFAC sanctions screening for the Spektrum platform, plus its Integration card and L4 evidence receipts. OFAC only publishes the SDN + Consolidated lists as bulk XML (no free per-name query API), so screening requires hosting the data and running name matching locally.
Deliverables
1. Standalone screening service (ofac-screening-service/, FastAPI/Python)
Hourly sync job: downloads SDN + Consolidated advanced XML exports; conditional GET (ETag/Last-Modified), sha256 fingerprint, atomic temp -> os.replace promote, parse + min-record validation before overwrite, timestamped archive, per-fetch provenance manifest.
POST /screen: normalized fuzzy name matching (RapidFuzz token_set_ratio) over primary + alias names; entity-type filter; lifespan startup + mtime hot-reload (no restart on new data).
Correctly parses the namespaced advanced schema (DistinctParty/SanctionsEntry) — the legacy sdnEntry path returns 0 records against SDN_ADVANCED.XML.
2. Integration card (Integration-Service)
integration_configs/compliance/ofac.json (screenEntity workflow -> POST {$base_url}/screen).
Registered in local_registry.json (ofac).
Transformation docs/ofac/is_ofac_screening_clear_transform.py -> criteria key isOfacScreeningClear (added to criteriakey-defaults.json, type compliance); fails closed.
3. L4 receipts
Each screen emits a canonical JSON+HTML receipt with content_hash (sha256 of the deterministic claim, excludes receipt_id/created_at) that feeds the Merkle tree, and a ready-to-attach evidence_item (source: automated, proofLevel: "4") for the unified-attestation-token.
A potential_match is never an auto-confirmed hit — surfaced for human adjudication.
Tests
Advanced-schema parser, fuzzy matcher (typos, reversed order, aliases, entity-type filter, clean-name clears), and receipt hashing (determinism, data-version sensitivity, L4 shape). 14/14 passing against a synthetic fixture (no 100MB download).
Follow-ups
Publish the transformation to spektrum-labs/Transformations/safeguards/<UUID>/ and replace the placeholder UUID in the card.
Natural next consumer: the ransom/incident token's existing ofacStatusOnTa field (Passed | Failed | Unknown).
